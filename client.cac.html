Low-level libca binding¶

Low level wrapper module over the libca interface.

class channel_access.client.cac.PV(name)¶

Create a new process value with name name.

This class encapsulates a channel id and forwards the callback functions to python.

Some of the API functionality is not implemented:

  • Multiple subscriptions

  • Non-callback get and put

  • Synchronous groups

The following keys can occur in a values dictionary:

value

Data value, type depends on the native type. For integer types and enum types this is int, for floating point types this is float. For string types this is bytes because there is no defined encoding.

status

Value status, one of Status.

severity

Value severity, one of Severity.

timestamp

An epics timestamp tule (seconds, nanoseconds). The time is counted from the epics epoch.

enum_strings

Tuple with the strings corresponding to the enumeration values. The entries are bytes because there is not defined encoding.

unit

String representing the physical unit of the value. The type is bytes because there is not defined encoding.

precision

Integer representing the number of relevant decimal places.

display_limits

A tuple (minimum, maximum) representing the range of values for a user interface.

control_limits

A tuple (minimum, maximum) representing the range of values accepted for a put request by the server.

warning_limits

A tuple (minimum, maximum). When the value lies outside of the range of values the status becomes Status.LOW or Status.HIGH.

alarm_limits

A tuple (minimum, maximum). When the value lies outside of the range of values the status becomes Status.LOLO or Status.HIHI.

clear_channel()¶

Destroy the channel of this PV.

This also removes a subscription. After this function has been called no other channel access function can be called.

create_channel()¶

Create a channel for this PV.

This functions must be called before any other channel access function can be called.

get(count, control, as_string)¶

Request data from the server.

This call does not return the data but only puts the request in the send buffer. If the data arrives the get_handler is called with the data.

Parameters

  • count (int) – The number of elements to request. If negative the number of elements of the PV is used.

  • control (bool) – If True the control properties (unit, limits, etc.) will be requested.

  • as_string (bool) – If True ask the server for a string representation instead of the native type.

put(value)¶

Request a value change on the server.

If this call returns the data is not changed it only puts the request for the change in the send buffer. If the response from the server is received the put_handler is called with the success status.

Parameters

value – The new value. For array PVs a list must be used.

put_handler¶

Handler called if the status for a put request arrives.

Signature: fn(succeeded)

Parameters

succeeded (bool) – If True the put request succeeded.

set_access_handler(access_handler)¶

Change the access rights handler to access_handler.

If the access rights change the handler is called with the new access rights.

subscribe(events, count, control, as_string)¶

Create a channel subscription.

If a subscription was already created it is first removed.

Parameters

  • events (Events) – Specifies which events will trigger the subscription.

  • count (int) – The number of elements to retrieve. If negative the number of elements of the PV is used.

  • control (bool) – If True the control properties (unit, limits, etc.) will be retrieved.

  • as_string (bool) – If True ask the server for a string representation instead of the native type.

channel_access.client.cac.attach_context()¶

Attach the current thread to the process wide ca context.

This must be called by all non-main threads who will use the channel access library.

channel_access.client.cac.detach_context()¶

Detach the current thread from the process wide ca context.

channel_access.client.cac.finalize()¶

Destroy the process wide ca context.

This should be called to shutdown the channel access library.

channel_access.client.cac.flush_io()¶

Flush outstanding IO requests to the server.

Create a process wide ca context.

This must be called by the main thread to initialize the channel access library.

Parameters

preemptive (bool) – If True preemptive callback mode is enabled.

channel_access.client.cac.pend_event(timeout)¶

Flush the send buffer and process CA background activity.

This function flushes the send buffer and then blocks for timeout seconds processing CA background activity.

Parameters

timeout (float) – Timeout interval in seconds. 0 means forever.

channel_access.client.cac.pend_io(timeout)¶

Flush the send buffer and wait for outstandig get requests.

This function flushes the send buffer and then blocks until outstanding get requests complete..

Parameters

timeout (float) – Timeout interval in seconds. 0 means forever.
